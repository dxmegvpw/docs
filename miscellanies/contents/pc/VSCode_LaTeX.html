My setup for VS Code and LaTeX

When I write notes or report assignments, I usually use LaTeX since it produces beautifully formatted mathematical equations, and above all, it is enjoyable to use.

To make our LaTeX lives happier, I usually use Visual Studio Code (VS Code) for a text editor. It helps us to write TeX code and introduce some features, like Grammarly or git etc., more easily. I cannot imagine my life without VS Code for me. In this notes, I will show you one of the installation and setup examples for LaTeX and VS Code. Our strategies are as follows:

  1. Install the Windows Subsystem for Linux (WSL) and VS Code.
    We will use a virtual environment and activate LaTeX on it.
  2. Log in to the VS Code account and synchronies the settings. (If you do not have a Microsoft account, you should create one as it will be easier to set up next time).
    Install TeX Live on Linux from the command prompt.

Install the application

The VS Code installer is available here. Confirm your Operating System and download appropriate versions to your computer.

Before (or after) installing, you should also install Ubuntu for your computer. Basically, we will be using the virtual environment and installing LaTeX (and Python, C++, etc.) on it from now on. You should open the Microsoft Store from the Start menu and find Ubuntu and install it on your PC.

When you have finished installing Ubuntu on your PC, let us start the setup for VS Code. Fortunately, we can synchronize the settings through the personal account, we can immediately finish several steps like finding the working folder, installing the application, etc. In summary, what you need to do at first is wait until the synchronization of the settings is finished and restart VS Code again to update the options.

Setting up the virtual environment and syncing with GitHub

Here is one of the most complex steps, installing Ubuntu and GitHub on VS Code. As I said, we can skip the setting about the VS Code through our account, but we should synchronize these applications on your computer.

Let me show you the step to use Ubuntu on the VS Code first. It is not so difficult, what you have to do is just to register your username and password on the Linux. When your turn on the Ubuntu for the first time, the registration form arise and you should follow its instruction. After finishing it, you should check the Remote Explorer tab of VS Code. If you successfully download the application "WSL" on the VS Code, you will open the workspace on the Linux. However, you will immediately notice that your Linux folder is almost empty. Therefore, let us install the GitHub to share the folders! Before installing GitHub, you should create a folder for it. In my case, I create the folder "git" under the user folder (e.g. /home/*username*/git). The preparation is done!

Now we need to install the git command on Linux. This is not a difficult step as the installer is already provided. All we need to do is type "sudo apt-get install git" at the command prompt. After waiting for the installation to complete, you can configure the git command. You should configure some information as follows

  • git config --global user.email "(your email address)"
  • git config --global user.name "(your username)"
This step is required when you commit the repostory. Once you have finished configuring, the next step is to clone the repository. You should be able to navigate to your github repository and find its URL. If you cannot get the URL, go back to the command prompt and run "git clone URL" to create a clone of the repository in your computer. That's all you need to do with GitHub.
    Useful tips:
  • If the commitments are conflicting, there are two solutions. One is to force your commitment. The command is "git push -f origin mian" (or equivalently "git push --force origin main"). The other way is to adjust the current repository. The commands are "git fetch origin main" and "git reset --hard origin/main".
